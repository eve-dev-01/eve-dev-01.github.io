EP Numerics


Standard Types

The EP Framework defines standard types of fixed size to guarantee that cross-platform code will always use the same data sizes.

int8, int16, int32, int64
uint8, uint16, uint32, uint64
float32, float64
char8, char16
Byte

The EP_Numerics.h file also provides some standard functions for Endian-swaps, alignment, and memory-size literals, such as _KiB. It also provides build-in values for the minimum and maximum values of the standard types.

Core Math

The EP_CoreMath.h file provides a set of basic core math functions. These include
- Clamp function
- Min/max function
- Square and cube
- Lerp, smooth lerp and smoother lerp
- Mapping value within range
- String to float and int

Bit Operations

The EP_Bitwise.h file provides a set of basic bitwise operations. These include
- Or, and, xor, not
- Left and right shift
- Low and high masking
- Bit setting and unsetting

Random

EP_Random.h provides an intended replacement for the std::rand() functionality. It implements a Mersenne twister with a very lightweight interface for accessing randomised values.

namespace ep
{
   // Seed the generator, must be called before any values are extracted
   void rand_seed(uint32);
   int32 rand32();
   int64 rand64();
   uint32 randu32();
   uint64 randu64();
   float32 randf32();
   float64 randf64();
}

Functions are provided for accessing floats, ints and uints in both 32- and 64-bit. The validity of the randomness of the 64 bit values is in question, so prefer 32-bit values.
